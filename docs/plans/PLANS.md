# Plans

Use this directory for long-running implementation plans.

A good plan is self-contained and executable by a future Codex session or human developer.

## ExecPlan template

Each ExecPlan should include:

- Goal
- Background
- Non-goals
- Current status
- Implementation steps
- Validation commands
- Acceptance criteria
- Risks
- User feedback required
- Progress log
- Decision log

## Rule

If a task requires more than one coherent patch, create or update an ExecPlan.
